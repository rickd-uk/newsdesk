Important deployment behavior
- push.sh does NOT copy the live SQLite DB.
- push-db.sh is for replacing the live DB with local articles.db.
- sudo password prompt during deploy is expected and intentionally kept.
- push.sh defaults to SSH host alias: hz-hel
- push.sh uses ssh -t so sudo prompts work during deploy

Live deployment issue encountered
- Service repeatedly failed with:
  - init users table: attempt to write a readonly database
- Root cause:
  - SQLite WAL mode needs write access to:
    - /opt/newsdesk
    - articles.db
    - articles.db-wal
    - articles.db-shm
- Manual fix that worked:
  - sudo systemctl stop newsdesk
  - sudo chown rick:www-data /opt/newsdesk
  - sudo chmod 775 /opt/newsdesk
  - sudo chown www-data:www-data articles.db
  - sudo chmod 664 articles.db
  - sudo chown www-data:www-data articles.db-wal articles.db-shm 2>/dev/null || true
  - sudo chmod 664 articles.db-wal articles.db-shm 2>/dev/null || true
  - sudo systemctl start newsdesk

2026-04-29/30 anonymous scroll fix
- User reported: when not logged in, the newsdesk feed could not scroll to see more articles.
- Investigation:
  - Anonymous users still receive paginated cards and the HTMX scroll sentinel.
  - The issue was client-side scroll locking, not server-side pagination/auth filtering.
  - static/app.js had multiple direct writes to document.body.style.overflow for auth modal, article modal, and filter drawer.
  - The filter panel could leave the body locked even when the page should be scrollable.
- Fix:
  - Added filterDrawerLocksPage() and updateBodyScrollLock() in static/app.js.
  - Modal/auth/filter close/open paths now call updateBodyScrollLock() instead of assigning overflow directly.
  - Body scroll only locks for:
    - open article modal
    - open auth modal
    - open mobile filter drawer at max-width 600px
  - Added resize listener so desktop/mobile transitions recompute the lock.
- Verification:
  - node --check static/app.js
  - env GOCACHE=/tmp/go-build-cache-newsdesk go test -tags fts5 ./...
- Note:
  - The repo already had many unrelated uncommitted changes before this fix.
  - The scroll fix only touched static/app.js.
